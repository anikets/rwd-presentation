Intro

Responsive Web Design (RWD) helps Web developers build apps which are cross device and cross platform compatible. Gone are the days of fixed 960 pixel width layouts. Long back. Today, Web Apps get accessed from a huge variety of devices. They differ in resolution. One thing common to all of them is the Web browser. Responsive Web Design (RWD) is the answer to target all these users with a single app which can be accessed through the browser.

RWD apps work on PC + Mac + Linux + Android + iOS + Windows Phone + everything else with a Web browser.

RWD offers an enormous benefit to lean startups by enabling them to target a large number of users, as compared to developing native apps whose reach is limited to the selected platform.

What are media queries?

Media queries are CSS statements that facilitate applying CSS conditionally to elements. Device properties like viewport width, orientation, number of colours the device can display and type of device can be accessed through these queries and used to write apply CSS rules conditionally.

A simple media query

@media (min-width: 700px) { ... }

This statement can be used in external CSS files, style tags in HTML files and in the link tag to used for including external stylesheets in HTML files.

A complex media query

@media (min-width: 700px), handheld and (orientation: landscape) {
  /* CSS rules here will be applied when:
     1. The viewport is 700px or wider.
     2. The device is a handheld and it's orientation
        is in landscape mode. */
}

Handling images

  • picture element specifies a way to load images according to viewport width.
  • srcset attribute also provides functionality similar to picture element.
  • WebP is a new image format from Google. File sizes are smaller compared to PNG and JPG. But it's not supported in Firefox currently.
  • SVG is useful for stuff like diagrams. They are resolution independent, and won't appear grainy or pixelated at any scale.
  • Font Icons are also resolution independent and allow styling of icons with CSS.
  • Progressive JPEG images appear blurred at the beginning and get sharper as the image gets loaded. They provide better user experience as compared to JPGs that get loaded from to bottom.

The picture element

<picture width="500"  height="500">
  <source  media="(min-width: 1000px)" src="large.jpg">
  <source  media="(min-width: 600px)" src="med.jpg">
  <source  src="small.jpg">
  <img  src="small.jpg" alt="">
  <p>Accessible  text</p>
</picture>
This is a work in progress and not currently supported in all browsers.

Word of caution

RWD is not a replacement for native mobile apps currently. Facebook and Linkedin regretted preferring RWD over native mobile apps. Browser based apps are not as high performant as native apps, and do not get access to device's native APIs. But if your app doesn't need them, well, who cares! Marginal performance is the tradeoff made to target large user base with RWD.
